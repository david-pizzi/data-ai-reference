Task Flows

Visual workspace feature for organizing and standardizing data project workflows
Module 6 Reference
Sections: Overview Key Concepts Task Types Creating Task Flows Knowledge Check
Overview
Visualize work flow — task flows provide a visual representation of the work flow within a workspace, making complex data projects easier to understand
Understand item relationships — see how different Fabric items (lakehouses, notebooks, pipelines, reports) connect and depend on each other
Standardize team work — establish consistent patterns and processes so team members follow the same workflow structure
Predefined templates — Fabric offers ready-made task flow templates for common scenarios, reducing setup time and enforcing best practices
Key Concepts
Task flow — a collection of connected tasks that represents an end-to-end data process; lives within a workspace and maps out how work gets done
Canvas — the workspace area where you visually arrange and connect tasks; each task flow has one or more canvases for organizing different phases
Task — a single unit of work within a task flow; represents one step in the process and can be linked to a specific Fabric item
Connector — logical arrows that link tasks together on the canvas; connectors show the sequence and dependencies between tasks but do not represent actual data flow

Creating Task Flows
Predefined end-to-end flows — select from built-in templates that cover common scenarios like lakehouse analytics, real-time streaming, or data warehouse projects
Create from scratch — build a custom task flow by adding tasks one at a time, connecting them with logical arrows to define your unique workflow
Import existing items — bring existing Fabric items from your workspace into a task flow to document and visualize current processes
Multiple canvases per workspace — organize complex projects across multiple canvases within a single workspace, each representing a different phase or team responsibility

Workspaces

Organizing, securing, and collaborating on Fabric items
Module 3 Reference
Sections: What Are Workspaces Workspace Roles Workspace Layout Workspace Settings Workspace Monitoring Best Practices Administration Knowledge Check
What Are Workspaces?
Collections of items — workspaces are collaborative containers where teams create, share, and manage Fabric items together
Item types — lakehouses, warehouses, reports, dashboards, notebooks, pipelines, dataflows, semantic models, and more all live inside workspaces
Capacity-backed execution — each workspace is assigned to a capacity (F, P, or trial) that provides the compute resources for running jobs and queries
Access management — workspaces provide role-based access control; assign users and groups to Admin, Member, Contributor, or Viewer roles to govern who can do what
Organizational boundary — workspaces map to teams, projects, or departments, providing a natural boundary for content organization and security
OneLake integration — every workspace automatically gets a corresponding folder in OneLake, making all workspace data accessible through the unified data lake
Workspaces replace the legacy “app workspaces” from Power BI. In Fabric, they extend to all workloads — not just BI content.

Workspace Layout

The workspace UI is organized into distinct regions for navigation, management, and content browsing.

Header bar — displays workspace name, search, and global navigation; quick access to settings and sharing options
Toolbar — contains actions like New, Upload, Filter, and View switcher; contextual buttons change based on selected items
Task flow (upper portion) — a visual workflow area that shows recommended steps or data processing sequences for the workspace
Items list — the main content area displaying all workspace items with columns for Name, Type, Owner, and Status
List view — default tabular layout showing items with sortable columns; ideal for managing many items efficiently
Lineage view — visual graph of data flow and dependencies between items; shows how data moves from sources through transformations to reports
Status column — shows refresh status, endorsement badges (Promoted/Certified), and sensitivity labels for each item at a glance
Workspace Settings

Configure workspace behavior, integrations, and storage through the settings panel.

General & Identity

General settings — workspace name, description, and image; set the workspace domain for governance alignment
Contact list — specify users or groups who receive notifications about workspace events and access requests
Workspace type — choose between Power BI workspace (BI-only) and Fabric workspace (full workload access); determines available item types

Integrations & Storage

Microsoft 365 integration — connect to a Microsoft 365 group for shared membership; link to SharePoint document library for file collaboration
Azure connections — connect Azure Data Lake Storage Gen2 for external data access; integrate Azure Log Analytics for monitoring and diagnostics
System storage — monitor and manage storage consumption; view per-item storage usage and clean up unused artifacts to stay within capacity limits
Git integration — connect workspace to Azure DevOps or GitHub for version control; sync workspace items with a Git branch for CI/CD workflows
Spark settings — configure default Spark pool, runtime version, and environment for notebooks and Spark jobs within the workspace
Workspace Monitoring

Track activity, job execution, and resource usage across workspace items.

Monitoring hub — centralized view of all running and recent activities across the workspace; filter by item type, status, or submitter
Item job event logs — detailed execution history for each item including start time, duration, status (success/failure), and error messages for troubleshooting
Workspace monitoring visualization — pre-built monitoring reports that surface trends in refresh frequency, duration patterns, and failure rates over time
Capacity metrics — understand how workspace activities consume capacity units (CUs); identify resource-intensive items and optimize scheduling
Audit logs — workspace events flow into the unified audit log for compliance; track who accessed, modified, or shared content and when
Best Practices
Set up task flows — define visual workflows in the workspace to guide team members through standard data processing sequences and reduce onboarding time
Pin frequently used workspaces — pin workspaces to the left navigation panel for quick access; reduces time spent searching across many workspaces
Use granular roles — assign the minimum role needed; prefer Viewer or Contributor over Member/Admin to follow the principle of least privilege
Create folders for organization — group related items into folders within the workspace (e.g., Raw, Curated, Reports) to keep content discoverable
Git integration for version control — connect workspaces to Git repositories for change tracking, code review, and safe deployment through branches
Maintain contact lists — keep the workspace contact list current so notifications about failures, access requests, and changes reach the right people
Workspace per environment — consider separate workspaces for Dev, Test, and Prod with deployment pipelines to promote content safely between stages
Administration

Tenant-level controls for managing workspaces across the organization.

Admin Portal

Workspace list — view all workspaces in the tenant; filter by state, capacity, or type; take admin actions on any workspace
Tenant settings — control who can create workspaces, which capacity types are available, and whether users can share content externally
Domain management — organize workspaces into business domains for governance boundaries and delegated administration

Auditing & Recovery

Auditing operations — track workspace structure changes through the unified audit log: CreateFolder, DeleteFolder, and UpdateFolder operations are captured automatically
Workspace recycle bin (preview) — deleted items are recoverable from the workspace recycle bin before permanent removal
Retention period — configurable from 7 to 90 days; admins set the default retention window for deleted workspace items across the tenant

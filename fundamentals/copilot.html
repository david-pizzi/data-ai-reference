Copilot & AI

Generative AI capabilities embedded across every Fabric workload
Module 5 Reference
Sections: Overview Copilot per Workload Data Engineering & Data Science Power BI Copilot Real-Time Intelligence Privacy, Security & Regions Enabling Copilot Knowledge Check
Overview
LLM-powered data analytics — Copilot uses large language models to help users work with data across the entire Fabric platform
Available across all workloads — integrated into Data Engineering, Data Science, Data Factory, Data Warehouse, Power BI, Real-Time Intelligence, and SQL Database
Enterprise developers — get intelligent code completion, error diagnostics, and performance insights in notebooks and SQL editors
Self-service users — generate dataflows, build pipelines, and create reports using natural language prompts
Business users — explore data with conversational Q&A, narrative summaries, and natural language queries
Augments, doesn’t replace — Copilot assists and accelerates human work but always keeps the user in control of decisions and outputs
SKU requirement — requires an F2 or higher Fabric SKU, or a Power BI Premium P SKU; not available on shared or trial capacities

Data Engineering & Data Science Copilot
Intelligent code completion — suggests code as you type in notebooks, including PySpark, SQL, and Scala, reducing boilerplate and accelerating development
Context-aware from workspace and lakehouse schemas — Copilot reads your lakehouse metadata, table schemas, and workspace context to generate relevant, accurate suggestions
Notebook-wide multi-step assistance — understands the full notebook context across cells, helping with multi-step data transformations and analysis workflows
Performance insights — provides recommendations on join strategies, data shuffles, partitioning, and caching to optimize Spark job execution
Fix with Copilot — when a cell errors, Copilot analyzes the stack trace and suggests a corrected version of the code with an explanation of what went wrong
/fix command — use targeted diagnostics by typing /fix in the Copilot chat to get focused error resolution for specific code blocks
Power BI Copilot
Report page suggestions — Copilot analyzes your semantic model and proposes complete report pages with relevant visuals, filters, and layout
Visual suggestions — describe the insight you want in natural language and Copilot generates the appropriate chart, table, or visualization
Narrative visual summaries — add a narrative visual that automatically generates plain-language summaries of the data shown on a report page
DAX generation and explanation — write measures and calculated columns by describing the business logic; Copilot generates DAX formulas and explains existing ones
App-scoped Copilot — embed Copilot into published Power BI apps so business users can ask questions directly within the app experience
Standalone cross-item experience — a dedicated Copilot pane that works across multiple semantic models and reports, enabling broad data exploration from a single entry point
Real-Time Intelligence Copilot
Natural language to KQL queries — describe what you want to find and Copilot translates your request into Kusto Query Language, handling complex joins, aggregations, and time-series operations
Dashboard tile editing with Copilot — modify Real-Time Dashboard tiles using natural language instructions to adjust queries, visuals, and filters without writing KQL directly
Modify and explore generated queries — Copilot-generated KQL is fully editable, allowing you to refine, extend, or learn from the generated code before executing
Privacy, Security & Regions
Azure OpenAI Service backing — all Copilot features are powered by Azure OpenAI Service, inheriting its enterprise security, compliance certifications, and responsible AI safeguards
Data processing regions — for tenants in the US and EU, data is processed within the same geographic boundary; prompts and responses do not leave the region
Cross-geo processing — tenants outside the US and EU may have data processed across geographic boundaries; this requires an explicit admin opt-in via tenant settings
Conversation history storage — Copilot retains conversation history for up to 28 days to support context continuity and improve the user experience
Users can clear chat history — individual users can delete their Copilot conversation history at any time from the Copilot pane
Sovereign clouds — Copilot in Fabric is not available for sovereign or government cloud environments (GCC, GCC High, DoD)
Enabling Copilot
Tenant admin settings required — Copilot is controlled through Fabric tenant settings in the Admin portal; it is not enabled by default and must be explicitly turned on
Enable for all or specific groups — admins can roll out Copilot to the entire organization or restrict it to specific security groups for a phased rollout
Azure OpenAI cross-geo setting — for tenants outside the US and EU, a separate toggle must be enabled to allow data to be processed by Azure OpenAI in other geographies
Copilot consumption billing — Copilot requests consume Fabric capacity units (CUs); usage is metered and billed against the workspace’s assigned capacity

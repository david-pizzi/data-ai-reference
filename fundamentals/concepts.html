What Is Microsoft Fabric?
End-to-end analytics SaaS platform — a single, unified environment that covers data movement, data science, real-time analytics, and business intelligence; all managed as a Software-as-a-Service offering
Replaces 5+ separate Azure services — consolidates Azure Data Factory, Azure Synapse Analytics, Azure Data Lake Storage, Azure Data Explorer, and Power BI into one integrated platform with a single billing model
OneLake as centralized data lake — all Fabric workloads store and access data in one logical lake, eliminating data silos and the need to move or duplicate data across services
Built on Azure Data Lake Storage Gen2 — OneLake uses ADLS Gen2 under the hood but abstracts away all infrastructure management; data is stored in open Delta-Parquet format by default
Single capacity model — one pool of compute capacity (measured in Capacity Units) powers all workloads; purchase once, use across all experiences
Deeply integrated with Microsoft 365 — accessible from Power BI, Teams, Excel, and SharePoint; data can be shared and governed across the Microsoft ecosystem
Fabric Architecture

Fabric is built as a layered SaaS platform. All workloads share the same foundation and platform services.

Top Layer — Workloads
Analytics Experiences
Data Factory · Data Engineering · Data Warehouse · Data Science · Real-Time Intelligence · Databases · Power BI · Fabric IQ · Industry Solutions
Middle Layer — Shared Services
Copilot, Governance & Platform
Copilot AI assistance · Microsoft Purview governance · OneLake catalog · Security & compliance · Admin portal · Git integration · APIs & SDKs · Monitoring hub
Foundation Layer — Storage
OneLake
Single logical data lake per tenant · Delta-Parquet format · Shortcuts for external data · Cross-cloud and cross-tenant sharing · Built on ADLS Gen2
All workloads read and write from the same OneLake storage. Shared services like Copilot, governance, and security are available across every workload without separate configuration.

Key Terminology

Essential terms for working with Microsoft Fabric, drawn from the official Fabric terminology documentation.

Term Definition
Capacity A dedicated set of resources (measured in Capacity Units) available for Fabric workloads. Capacity determines performance, throttling limits, and billing.
Tenant A dedicated instance of Fabric for an organization, tied to an Entra ID (Azure AD) tenant. Each tenant has exactly one OneLake.
Workspace A logical container for organizing Fabric items. Workspaces map to teams or projects and support role-based access control (Admin, Member, Contributor, Viewer).
Item Any artifact created in Fabric: lakehouse, notebook, pipeline, warehouse, report, semantic model, etc. Items are the building blocks of analytics solutions.
Workload / Experience A distinct set of capabilities in Fabric (e.g., Data Engineering, Data Factory, Power BI). Each workload provides item types tailored to a specific analytics task.
Lakehouse A data architecture combining the flexibility of a data lake with the querying power of a data warehouse. Stores data in Delta-Parquet format with a SQL analytics endpoint.
Notebook An interactive coding environment for data exploration, transformation, and machine learning using PySpark, Scala, SparkSQL, or SparkR.
Pipeline A Data Factory orchestration workflow that coordinates data movement and transformation activities in a defined sequence with scheduling and monitoring.
Dataflow Gen2 A low-code data transformation tool using Power Query Online. Connects to 200+ sources and loads transformed data into Fabric destinations.
Warehouse A fully managed, T-SQL-compatible data warehouse in Fabric. Supports cross-database queries and automatic performance optimization.
SQL Analytics Endpoint An auto-generated, read-only T-SQL endpoint on a lakehouse or mirrored database, enabling SQL queries without data movement.
Experiment A Data Science item for organizing and tracking ML model training runs using MLflow.
Model A trained machine learning model registered in Fabric’s model registry for versioning, scoring, and deployment.
V-Order A write-time optimization applied to Parquet files that improves read performance across all Fabric engines without additional tuning.
Shortcut A reference to data stored externally (ADLS Gen2, Amazon S3, Google Cloud Storage, Dataverse) that appears in OneLake without copying or moving data.
OneLake Fabric’s built-in, hierarchical data lake. One per tenant, organized into workspaces and lakehouses. All compute engines read and write to OneLake.
Eventhouse A Real-Time Intelligence item optimized for storing and querying large volumes of time-series and streaming data using KQL.
Eventstream A no-code experience for capturing, transforming, and routing real-time events from sources like Azure Event Hubs, IoT Hub, and custom apps.
KQL Database A database within an Eventhouse that stores data optimized for Kusto Query Language (KQL) queries and time-series analysis.
Ontology A Fabric IQ item that maps business concepts (entities, relationships) to underlying data assets, enabling AI agents to reason in business terms.
OneLake Architecture

OneLake is Fabric’s built-in data lake — think of it as “OneDrive for data.” Every tenant gets exactly one OneLake, automatically provisioned.

Hierarchy

Tenant — the top-level boundary; each organization has one OneLake instance
Workspaces — logical containers within the tenant, each with its own access control and item collection
Lakehouses — within workspaces, lakehouses contain Tables/ (Delta-Parquet) and Files/ (any format) directories
All engines share storage — Data Engineering, Data Warehouse, Power BI, and Data Science all read/write to the same OneLake files

Key Capabilities

Delta Parquet format — all tabular data stored in open Delta Lake format with Parquet files; supports ACID transactions, time travel, and schema evolution
Shortcuts for zero-copy access — reference external data in ADLS Gen2, Amazon S3, Google Cloud Storage, or Dataverse without moving or duplicating it
Cross-tenant data sharing — securely share data across organizational boundaries while maintaining governance and access controls
OneLake file explorer — access OneLake data directly from Windows File Explorer, just like OneDrive
Real-Time Hub

The Real-Time hub is a unified SaaS experience for discovering, managing, and consuming streaming data across your organization.

Centralized streaming catalog — a single location to discover all streams and KQL database tables available in your Fabric tenant
Three stream categories — My streams (streams you created), All streams (streams shared across the organization), and Microsoft sources (Azure Event Hubs, IoT Hub, Azure SQL DB change data, Blob Storage events)
No-code integration — connect to data sources and set up eventstreams with transformations directly from the hub, without writing code
Set alerts and actions — configure Data Activator reflexes to monitor stream conditions and trigger notifications or Power Automate flows
Feeds into KQL databases — streaming data lands in Eventhouse KQL databases for real-time querying and dashboard visualization
The Real-Time hub replaces the need to manually wire up event ingestion; all streaming data sources are browsable and actionable from one place.

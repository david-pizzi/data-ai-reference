Lakehouse vs Warehouse
Primary engine — Lakehouse uses Apache Spark; Warehouse uses T-SQL engine
Multi-table transactions — Warehouse supports full ACID transactions across tables; Lakehouse supports single-table ACID via Delta
Data complexity — Lakehouse handles structured, semi-structured, and unstructured data; Warehouse is optimized for structured data only

Scenarios
Enterprise BI migration — use Warehouse with Copy Activity to migrate existing SQL-based data warehouse workloads; leverage T-SQL skills and dimensional models
IoT telemetry dashboard — use Eventhouse with Eventstream for real-time ingestion from IoT devices; KQL for fast aggregation and alerting
Data science exploration — use Lakehouse with Spark notebooks to explore semi-structured data, train ML models, and iterate quickly
Self-service analytics — use Dataflow Gen2 for citizen developers to prepare and transform data without writing code; publish to Warehouse or Lakehouse
Multi-model application — use Cosmos DB (mirrored) for operational NoSQL data with vector search for AI features, mirrored to OneLake for analytics

Lifecycle Overview

Data in Microsoft Fabric flows through six stages — all data stays in OneLake throughout, with no movement between engines.

Get Data ▶ Store Data ▶ Prepare & Transform ▶ Analyze & Train ▶ Track & Visualize ▶ External Integration
All data stays in OneLake — every stage reads from and writes to the same centralized data lake in open Delta-Parquet format
No data movement between engines — Data Engineering, Data Warehouse, Power BI, Real-Time Intelligence, and Data Science all access the same physical data
Unified governance — security, lineage, and access policies apply consistently across the entire lifecycle
Not strictly linear — stages can be iterative; data may loop back through preparation and analysis multiple times
Stage 1 — Get Data

Bring data into OneLake from any source — real-time, batch, replicated, or virtualized.

Eventstreams — real-time ingestion from IoT devices, apps, and services; captures streaming events into Eventhouse or Lakehouse
Data pipelines — batch and scheduled ingestion with 200+ connectors (databases, SaaS apps, files, APIs); orchestration with copy activities
Mirroring — continuous replication from operational databases (Azure SQL, Cosmos DB, Snowflake, PostgreSQL, MySQL) into OneLake with near real-time sync
Shortcuts — zero-copy data virtualization; reference data in ADLS Gen2, Amazon S3, Google Cloud Storage, or Dataverse without moving it
Copy job — simplified one-click data movement for common copy scenarios; less complex than full pipeline orchestration
OneLake data sharing — cross-tenant and cross-workspace data access; share data with external organizations without duplication
Stage 2 — Store Data

Choose the right storage destination based on workload type, query pattern, and latency requirements.

Lakehouse — flexible big data storage combining Files and Delta tables; auto-generates a SQL analytics endpoint for querying without a separate warehouse
Warehouse — full T-SQL data warehouse with stored procedures, views, and ACID transactions; ideal for traditional BI and complex joins
Eventhouse — real-time analytics store optimized for telemetry, logs, and IoT data; query with KQL (Kusto Query Language)
SQL Database — transactional OLTP workloads; auto-mirrors data to OneLake so analytics engines can read it directly
Semantic models — business logic layer with measures, hierarchies, and relationships; the single source of truth for reporting
OneLake shortcuts — zero-copy references to external or cross-workspace data; no storage duplication

Storage Comparison

Store Best For Query Language Format Key Feature
Lakehouse Big data, data engineering Spark SQL, T-SQL (endpoint) Delta-Parquet + Files Auto SQL endpoint

Stage 3 — Prepare & Transform

Clean, enrich, and reshape data using low-code or pro-code tools — all writing back to OneLake.

Dataflow Gen2
Low-code Power Query experience with 300+ transformations. Visual interface for merging, filtering, pivoting, and shaping data. Outputs directly to Lakehouse, Warehouse, or KQL Database.
Notebooks
Jupyter-style notebooks with Python, T-SQL, Scala, and R support. Spark DataFrames for distributed processing. Ideal for complex transformations, ML feature engineering, and ad-hoc exploration.
User Data Functions
Reusable Python functions deployed as REST endpoints. Lightweight, serverless compute for on-demand transformations. Can be called from pipelines, notebooks, or external applications.
Stage 4 — Analyze & Train

Extract insights, build models, and enable intelligent experiences on top of your governed data.

Data Agents — conversational Q&A over enterprise data; users ask natural language questions and agents translate to structured queries across lakehouses, warehouses, and semantic models
Operations Agents — autonomous monitoring and action; detect anomalies, trigger alerts, and execute remediation workflows without manual intervention
MLflow experiments — automatic logging of training runs, parameters, metrics, and artifacts; compare models side by side
ML models — MLflow model registry with versioning, staging, and production deployment; integrated with Spark and Python notebooks
GraphQL APIs — flexible data access layer exposing Fabric data through a single endpoint; clients request exactly the fields they need
Copilot in Power BI — natural language analytics, auto-generated visuals, narrative summaries, and DAX formula suggestions powered by AI
Stage 5 — Track & Visualize

Present data as actionable insights through reports, dashboards, and automated responses.

Power BI reports — interactive visualizations built on semantic models; DirectLake mode for sub-second performance directly from OneLake without data import
Translytical taskflows — take action directly from reports; trigger operational workflows, update records, or initiate processes without leaving the analytics experience
Real-Time dashboards — sub-second latency visualizations from Eventhouse and KQL databases; auto-refreshing tiles for live monitoring
Activator — condition-based automated actions; set rules on data streams or semantic models and trigger alerts, emails, Power Automate flows, or Teams messages
Anomaly detection — adaptive pattern identification in time-series data; automatically surfaces unexpected changes and deviations
Stage 6 — External Integration

Connect Fabric to the broader Microsoft ecosystem and third-party services.

Power Automate — workflow automation triggered by Fabric events, Activator alerts, or scheduled actions; 1,000+ connectors to external systems
Microsoft 365 — embed reports in Teams channels, export to Excel and PowerPoint, share via SharePoint; data stays governed in OneLake
REST APIs & client libraries — programmatic access to Fabric items, workspaces, and data; Python, .NET, and JavaScript SDKs
Microsoft Entra ID — unified authentication and authorization; single sign-on, conditional access, and role-based permissions across all Fabric workloads
Git integration — connect workspaces to Azure DevOps or GitHub repositories; version control for pipelines, notebooks, semantic models, and reports
Microsoft Purview — enterprise governance with data cataloging, sensitivity labels, lineage tracking, and compliance policies across OneLake

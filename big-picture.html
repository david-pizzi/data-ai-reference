The Core Shift: Fragmented → Unified
Study: Introduction to Fabric →
Microsoft Fabric replaces 5+ separate Azure services with a single SaaS platform. One portal, one billing model, one security boundary — all workloads share the same data in OneLake.
Three-Layer Architecture
Study: OneLake & Lakehouses →
Fabric is built in three layers: workloads at the top, platform services in the middle, and OneLake as the universal storage foundation. Governance wraps everything.
Two Compute Paradigms, One Storage
Study: Lakehouse & Warehouse →
Lakehouse and Warehouse are different compute engines optimized for different workloads — but both read and write Delta tables on the same OneLake storage. Choose by workload, not by data.
Governance Hierarchy
Study: Administration & Governance →
Everything in Fabric lives inside a strict hierarchy: Tenant → Capacity → Workspace → Items. Admin settings cascade down. Purview adds sensitivity labels across all levels.

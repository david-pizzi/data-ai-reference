Goal for This Phase
Understand what Fabric is — a unified SaaS analytics platform that replaces Microsoft's fragmented data stack
Understand why Microsoft built it — customers needed one platform instead of stitching together ADF, Synapse, ADLS, Power BI, and ADX
Know how it replaces previous services — each legacy Azure analytics service maps to a Fabric workload
See where it’s heading — Fabric IQ adds an ontology/semantic layer, turning a data platform into an intelligence platform

The Story So Far — Why Fabric Exists

Before Fabric, Microsoft’s analytics stack was fragmented across multiple services:

ServiceRoleProblem
Azure Data FactoryData ingestion & orchestrationSeparate portal, separate billing, separate RBAC
Azure Synapse AnalyticsData warehousing & SparkComplex provisioning, multiple pool types, steep learning curve
Azure Data Lake StorageRaw data storage (ADLS Gen2)Just storage — no compute, no governance built in
Power BIReporting & visualisationDisconnected from engineering workflows, separate capacity model
Azure Data ExplorerReal-time analytics (KQL)Niche service, not integrated with warehouse or lake
The result: customers managed 5+ portals, 5+ billing models, 5+ security boundaries — and still needed to build ETL between them
Fabric’s answer: unify everything into a single SaaS experience on OneLake — one storage layer, one security model, one billing model

Exploration — Navigate the Trial Environment

Activate a Fabric trial, create a workspace, and explore these item types:

  • Activate trial — go to app.fabric.microsoft.com, start a 60-day free trial
  • Create a workspace — name it something like "Phase1-Sandbox", assign trial capacity
  • Lakehouse — create one, upload a CSV, see it appear as a Delta table, query via SQL endpoint
  • Warehouse — create one, run a T-SQL CREATE TABLE, compare the experience to Lakehouse
  • Notebook — open a Spark notebook, connect to your lakehouse, run a simple PySpark query
  • Pipeline — create a Data Factory pipeline, add a Copy activity (even a no-op), see the orchestration UI
  • Report — create a Power BI report from the lakehouse SQL endpoint’s default semantic model
  • Eventhouse — create one, see the KQL database experience (Real-Time Intelligence)
  • Ontology — if available in preview, explore the IQ ontology item type

4 Principles to Internalise

Carry these through every phase of study. They are the mental model for how Fabric works.

Principle 1
OneLake = One Copy of Data
All workloads read and write from the same storage layer. No ETL between Fabric services. Shortcuts bring in external data without duplication. This is the single biggest architectural shift from the legacy stack.
Principle 2
SaaS, Not PaaS
No infrastructure to provision, no clusters to manage, no Azure subscription required. Capacity-based billing (CUs) that scales up/down or pauses. Microsoft manages the platform so customers focus on data.
Principle 3
Delta Lake Is the Universal Format
Every table in OneLake is stored as Delta Lake (Parquet + transaction log). Open format readable by any engine — Spark, T-SQL, Python, Power BI. No vendor lock-in on the data layer.
Principle 4
Fabric IQ Is the Next Frontier
IQ adds a semantic/ontology layer on top of data. Business concepts, rules, and relationships are modelled as a labelled property graph. This powers AI agents that understand the business, not just the data.

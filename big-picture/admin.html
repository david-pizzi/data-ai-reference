Fabric Architecture
Software-as-a-Service platform — provides a simple and integrated approach while reducing administrative overhead
All-in-one analytics solution — covers data warehousing, data engineering, data integration, data science, real-time intelligence, and business intelligence
Built on OneLake — all data stored in OneLake, built on Azure Data Lake Storage (ADLS) Gen2 architecture
One OneLake per tenant — hierarchical in nature, provides a single-pane-of-glass file-system namespace spanning users, regions, and clouds
Fabric architecture — OneLake as the foundation with each experience built on top
Core Fabric Concepts
ConceptDescription
TenantDedicated space for organizations to create, store, and manage Fabric items. Aligned with Microsoft Entra ID. Maps to the root of OneLake — top level of the hierarchy.
CapacityDedicated set of resources available at a given time. Defines the ability of a resource to perform an activity or produce output. A tenant can have one or more capacities. Offered through the Fabric SKU and Trials.
DomainLogical grouping of workspaces. Used to organize items in a way that makes sense for the organization (e.g., sales, marketing, finance).
WorkspaceCollection of items that brings together different functionality in a single tenant. Acts as a container that uses capacity and provides access controls.
ItemsBuilding blocks of the Fabric platform — data warehouses, data pipelines, datasets, reports, dashboards, and more.

Admin Role & Tasks

Several roles work together to administer Microsoft Fabric: Microsoft 365 admin, Power Platform admin, Fabric capacity admin. The Fabric admin role was formerly known as Power BI admin.

Task AreaDescription
Security & Access ControlManage RBAC to define who can view/edit content. Set up data gateways for on-premises data sources. Manage user access with Microsoft Entra ID.
Data GovernanceSecure inbound/outbound connectivity. Monitor usage and performance metrics. Apply data governance policies to restrict unauthorized access.
Customization & ConfigurationConfigure private links, define data classification policies, adjust report and dashboard appearance.
Monitoring & OptimizationConfigure monitoring and alerting. Optimize query performance. Manage capacity and scaling. Troubleshoot data refresh and connectivity issues.

Admin Tools

Fabric admins can perform most admin tasks using one or more of the following tools:

Fabric Admin Portal
Web-based portal to manage all aspects of the platform
• Centrally manage, review, and apply settings for entire tenant or by capacity
• Manage users, admins, groups, audit logs
• Monitor usage and performance
• Noteworthy: Fabric on/off switch in tenant settings
PowerShell Cmdlets
Automate common administrative tasks via PowerShell
• Create and manage groups
• Configure data sources and gateways
• Monitor usage and performance
• Manage Fabric admin APIs and SDKs
Admin APIs & SDKs
Programmatic interaction with Fabric via REST APIs
• Automate admin tasks and integrate with other systems
• Any HTTP client with OAuth 2.0 (e.g., Postman)
• PowerShell scripts for automation
Admin Monitoring Workspace
Built-in workspace for Fabric tenant admins
• Feature Usage and Adoption dataset and report
• Insights on usage and performance
• Identify trends, patterns, and troubleshoot issues
• Can be shared with other users
Tenant settings in the Fabric admin portal — Fabric on/off switch for delegation
Feature Usage and Adoption report in the Admin monitoring workspace

Manage Fabric Security

As a Fabric admin, part of your role is to manage security for the Fabric environment, including managing users and groups, and how users share and distribute content.

User License Management
User licenses control the level of user access and functionality within the Fabric environment
Cost control — manage and allocate licenses efficiently to prevent unnecessary expenses
Compliance — limit access to sensitive data and ensure compliance with data protection laws and regulations
Managed in Microsoft 365 admin center — license type in workspace settings relates to the user license type
Items & Sharing
Workspace apps — items are best distributed through a workspace app or the workspace directly
Least permissive rights — the first step in securing data; share read-only app for report access
Collaboration — grant workspace access for collaboration and development
Internal & external sharing — manage distribution both internally and outside the organization in compliance with policies

Govern Data in Fabric

Fabric includes built-in governance features to help you manage and control your data:

Endorsement
Key governance feature — builds trust in data assets by marking Fabric items as reviewed and approved
Promoted — workspace members with contributor or admin role can promote content; appears with a Promoted badge
Certified — requires a formal review by a designated reviewer; appears with a Certified badge; admins manage the process
Request certification — non-admins can request certification via the item's More menu in the Fabric portal
Metadata Scanning & Scanner API
Admin REST APIs to scan Fabric items for sensitive data
• Scan data warehouses, pipelines, datasets, reports, dashboards
• Works on both structured and unstructured data
• Must be set up by an Admin before use
Data Lineage & Impact Analysis
Track the flow of data through Fabric
• See where data comes from, how it's transformed, and where it goes
• Lineage view in workspaces shows data usage
Microsoft Purview Hub
Purview hub (preview) — manage and govern your Fabric data estate from within the Fabric portal
Reports — insights about sensitive data, item endorsement, and domains
Gateway to Purview portal — Data Catalog, Information Protection, Data Loss Prevention, and Audit

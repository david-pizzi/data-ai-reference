Phase 4 Goal
Understand how business users consume Fabric data through Power BI and semantic models — the final mile from data engineering to business insight
You don't need DAX expertise — but you need the concepts because semantic models are the bridge between data engineering and Fabric IQ
Focus on the architecture: how semantic models sit on top of lakehouses and warehouses, how DirectLake mode connects them, and how they feed into ontologies
By the end of Phase 4 you should be able to explain to a customer how their data flows from OneLake through a semantic model into Power BI reports — and from there into an IQ ontology
Why Semantic Models Matter
Central to the Fabric IQ story: ontologies can be generated directly from Power BI semantic models, inheriting entity definitions, relationships, and measures
Understanding semantic models = essential for understanding IQ. The ontology layer doesn't start from scratch — it builds on what already exists in your semantic models
Business-friendly abstraction: semantic models translate raw tables and columns into business terms (Customer, Revenue, Region) that non-technical users can understand
Single source of truth: one semantic model can serve multiple reports, ensuring consistent definitions of KPIs and measures across the organisation
Security boundary: Row-Level Security (RLS) and Object-Level Security (OLS) are defined in the semantic model, not in individual reports

Key Concepts
ConceptDetail
Semantic Model A curated analytical model containing measures, KPIs, relationships, and hierarchies. The business-friendly layer on top of raw data — defines what “Revenue” or “Active Customer” actually means across the organisation.
DAX (Conceptual) Data Analysis Expressions — the formula language for measures and calculations in semantic models. Understand what it does (aggregations, time intelligence, filtering context), not the syntax — that comes with practice.
Star Schema Fact tables (events — sales, clicks, transactions) surrounded by dimension tables (context — date, customer, product). The standard pattern for semantic models. Maps directly to ontology entity types in IQ.
Semantic Model → Ontology The bridge to IQ. Fabric converts tables → entity types, relationships → graph edges, measures → properties. This is how business logic flows from Power BI into the knowledge graph and AI agents.

Hands-On Exercise

This exercise connects Phase 4 back to Phase 2 and Phase 3, completing the end-to-end Fabric flow.

Step 1 — Create a semantic model: take the lakehouse or warehouse tables from your Phase 3 work and build a basic semantic model with star schema relationships
Step 2 — Build a Power BI report: create a simple report on top of the semantic model — a few visuals showing key metrics. Use DirectLake mode to connect directly to OneLake data without import or DirectQuery overhead
Step 3 — Generate an ontology: from the semantic model, generate an IQ ontology — see how tables become entity types, relationships become graph edges, and measures become properties
Step 4 — Connect a data agent: use the ontology from Step 3 to power a data agent, completing the loop from Phase 2
End-to-end flow: Data (Phase 3) → Semantic Model → Ontology (Phase 2) → Data Agent — this is the single most valuable exercise in the entire plan

What is Microsoft Fabric?
All-in-one SaaS analytics platform covering data ingestion, transformation, storage, real-time processing, data science, and business intelligence
Single platform replaces Azure Data Factory + Synapse Analytics + Power BI + Azure Data Lake Storage + Azure Data Explorer under one unified experience
OneLake is the unified storage layer — one logical data lake per tenant, built on ADLS Gen2, accessible by all workloads with zero-copy
SaaS model — no infrastructure to manage, no Azure subscription required, capacity-based licensing (CU)
Built-in governance via Microsoft Purview — sensitivity labels, access control, and auditing applied across all workloads automatically
Copilot embedded across workloads — assists with query authoring, pipeline building, code generation, and data exploration

Platform Architecture

Three-layer model: workloads on top, shared platform services in the middle, OneLake storage at the foundation.

LayerWhat It DoesKey Components
WorkloadsRole-specific analytics experiences — each optimised for a persona (engineer, analyst, scientist)Data Engineering, Data Factory, Data Warehouse, Data Science, Power BI, Real-Time Intelligence, Databases, Industry Solutions, IQ
Platform ServicesShared capabilities used by all workloadsCopilot, Governance (Purview), OneLake Catalog, Security, Admin
OneLakeUnified logical data lake — single namespace per tenantADLS Gen2 underneath, Delta/Parquet format, Shortcuts for external data
Data mesh architecture: each business domain manages its own data products in workspaces, OneLake provides the unified storage layer
OneLake Catalog: discover, explore, and govern all Fabric items across workspaces and domains from one central hub

The 9 Workloads

Each workload has its own tools, APIs, and UX — but all share OneLake storage and platform services.

Power BI
Interactive dashboards, reports, semantic models. The reporting layer for all Fabric data.
Data Factory
Data integration — 200+ connectors, Dataflows Gen2 (Power Query), orchestration pipelines.
Data Engineering
Apache Spark notebooks & jobs for large-scale data processing. PySpark, Scala, SparkSQL.
Data Warehouse
Full T-SQL transactional warehouse. Compute/storage separation. Delta Lake format natively.
Data Science
ML experiments, model training, MLflow tracking, model registry. Integrates with Azure ML.
Real-Time Intelligence
Streaming ingestion, Eventhouse (KQL), real-time dashboards. IoT, logs, clickstreams.
Databases
Operational SQL databases in Fabric. Mirroring from Azure SQL, Cosmos DB, Snowflake, Databricks.
Industry Solutions
Pre-built data models and solutions for specific industries (healthcare, retail, etc.).
IQ (Preview)
Business semantics — ontology layer, labelled property graph, AI-ready unified semantic model.

OneLake — The Unified Data Lake
ConceptDetail
What it isSingle logical data lake per tenant — like OneDrive for data. Built on ADLS Gen2. No separate provisioning.
HierarchyTenant → Workspaces → Lakehouses / Warehouses / KQL DBs. Each workspace is a folder in OneLake.
FormatDelta Lake (Parquet + transaction log) for all tables. Open format, readable by any engine.
ShortcutsMount external storage (ADLS, S3, GCS, Dataverse) without copying data. Virtual access, cached for performance.
Zero-copyAll workloads read/write the same data in OneLake — no ETL between Fabric services.
No Azure neededFabric is SaaS — OneLake comes with the tenant. No storage accounts, resource groups, or RBAC to configure.
Think of it as: "OneDrive for analytics" — every Fabric user automatically has access to their organisation's data lake

Licensing & Capacity
ConceptDetail
Capacity Units (CU)Fabric is billed by capacity — a pool of compute resources shared across all workloads. Measured in CUs.
SKUsF2, F4, F8 … F2048. Each SKU provides a fixed number of CUs. Scales up/down or pauses.
Pay-as-you-goAzure subscription billing per second of CU consumption. Pause when not in use.
Reservations1-year or 3-year committed pricing for significant discount (like Azure Reserved Instances).
Power BI Pro/PPUStill needed for content sharing. PPU adds premium features per user. Fabric capacity replaces P-SKUs.
Free trial60-day Fabric trial capacity available per user — great for POCs. Includes all workloads.
Key shift: Fabric F-SKUs replace the old Power BI Premium P-SKUs — same capacity model but unlocks all Fabric workloads, not just Power BI

Fabric IQ (Preview) — The Semantic Layer
Ontology: new item type that models core business concepts, rules, and relationships into a unified semantic layer
Labelled property graph: connects to OneLake data and existing semantic models to create a live, structured model of how the business operates
AI-ready: powers consistent decisions, reusable metrics, and context-aware automation across Fabric
Data agents + operations agents: IQ enables AI agents that understand the business ontology and can reason over the data
Why it matters: IQ is where Fabric is heading — the next evolution beyond lakehouse/warehouse into semantic AI

Fabric vs Legacy Azure Analytics Services
Legacy ServiceWhat Fabric ReplacesMigration Story
Azure Synapse AnalyticsSpark pools, SQL pools, pipelines, Synapse StudioFabric is the successor. Synapse features are now Fabric workloads. No new Synapse investments.
Azure Data FactoryData integration and orchestrationADF v2 continues but Fabric Data Factory is the modern path. Pipelines are nearly identical.
Azure Data Lake StorageData lake storageOneLake replaces standalone ADLS. Use Shortcuts to access existing ADLS without migration.
Azure Data ExplorerReal-time analytics, KQLReal-Time Intelligence in Fabric. Existing ADX clusters can be linked.
Power BI PremiumPremium capacity (P-SKUs)Fabric F-SKUs replace P-SKUs. F-SKUs unlock all Fabric workloads + Power BI Premium features.
Azure Analysis ServicesSemantic models / tabular modelsPower BI semantic models in Fabric. AAS is in maintenance mode.
Key message for customers: Fabric unifies what used to be 6+ separate Azure services into one SaaS platform — simpler to manage, govern, and bill

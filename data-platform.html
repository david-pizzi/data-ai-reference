Goal
Build working knowledge of core data platform components — how data flows into, through, and out of Fabric. Focus on understanding patterns and when to use each component, not memorising syntax.

Key Patterns
Medallion Architecture: Bronze (raw) → Silver (cleansed) → Gold (business-ready). Standard Fabric data engineering pattern. Each layer adds quality and structure. Bronze is append-only ingestion, Silver applies schema and deduplication, Gold is aggregated and optimised for consumption.
Lakehouse vs Warehouse: Lakehouses are file-based (Delta) with Spark + SQL. Warehouses are SQL-native with T-SQL. Both read from OneLake. Choose Lakehouse for data engineering and mixed workloads (Python, Spark, SQL). Choose Warehouse for pure SQL analytics with familiar T-SQL patterns.
Pipelines vs Dataflows vs Notebooks: Pipelines orchestrate workflows — they sequence and schedule activities but do not transform data themselves. Dataflows Gen2 are low-code Power Query transformations ideal for business analysts. Notebooks are code-first PySpark/SQL for engineers and data scientists.
DirectLake: Power BI reads Delta tables directly from OneLake without import or DirectQuery. Game-changer for performance — combines the speed of import mode with the freshness of DirectQuery. No data duplication, no scheduled refresh, sub-second queries on large datasets.

RTI feeds into Fabric IQ: Operations Agent monitors Eventstream data, Eventhouses feed ontology bindings. Full pipeline: Eventstream → Eventhouse → KQL Database → Ontology → Operations Agent.
Key components: Eventstream ingests real-time data (IoT, logs, clickstreams). Eventhouse stores and queries via KQL. Real-time dashboards auto-refresh. Activator triggers actions on conditions.

Hands-On Exercise

Build a simple end-to-end flow in your Fabric trial to see the full picture from ingestion to report.

Step 1 — Ingest: Create a Lakehouse in your workspace. Use a Data Factory pipeline to ingest a sample dataset (e.g., NYC Taxi or AdventureWorks) into the Lakehouse Files section.
Step 2 — Transform: Open a Spark notebook (use built-in samples). Read the raw files, apply basic transformations (filter, rename columns, add calculated fields), and write the results as Delta tables in the Lakehouse Tables section.
Step 3 — Model: Open the Lakehouse SQL analytics endpoint. Verify your Delta tables appear as SQL tables. Explore the auto-generated semantic model.
Step 4 — Report: Connect Power BI to the Lakehouse using DirectLake mode. Build a simple report with a few visuals. Notice the performance — no import, no refresh, direct reads from OneLake.
Full picture: Pipeline (ingest) → Notebook (transform) → Delta tables (store) → DirectLake (connect) → Power BI (report). This is the core Fabric data platform pattern.
